## 2. 시나리오 (5~10줄)
이 프로그램은 대학생이 수강 과목을 등록하고 학업 전반을 체계적으로 관리할 수 있는 통합 학습 도우미 시스템입니다.
사용자는 과목명, 학점 수, 담당 교수, 현재까지의 점수를 입력하면, 시스템이 자동으로 현재 평균 점수와 지난 학기 대비 성적 변화를 계산하여 보여줍니다. 이를 통해 내 성적이 올랐는지 내렸는지 한눈에 파악할 수 있습니다.
점수 구간에 따라 A~F 학점을 판별하고, 특정 조건(예: 평균 90점 이상이면서 학점이 3학점 이상)을 만족하는 과목에는 '우수 과목' 배지를 부여하는 특별 로직도 포함됩니다.
나아가 과제 마감일, 시험 일정, 할 일(To-Do) 목록을 기록하고 조회하는 기능을 추가하여, 단순 성적 계산기를 넘어 학업 전체를 관리하는 스마트 플래너로 발전시킬 계획입니다. 최종적으로는 메뉴 시스템과 파일 저장 기능을 통해 언제든 내 학습 현황을 꺼내볼 수 있는 완성도 높은 프로그램으로 완성할 예정입니다.

## 🟦 [1차 과제: V1.0] 시나리오 기획
    ✨ V1.0 개발 내용

사용자로부터 과목명(문자열), 학점 수(정수), 담당 교수 이니셜(문자), 현재 점수(실수), 지난 학기 평균(실수)을 입력받습니다.
사칙연산을 활용해 현재 평균 점수와 지난 학기 대비 점수 차이를 계산하여 출력합니다.
f-string을 사용하여 입력 정보와 계산 결과를 깔끔하게 출력합니다.
사용된 변수: subject_name(str), credits(int), professor_initial(str), current_score(float), last_avg(float), score_diff(float) — 총 6개 변수, 3가지 자료형

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정

### ✨ V2.0 개발 내용

- `for`문 + `range()`로 여러 과목을 입력받아 3개 리스트(`subject_names`, `scores`, `grades`)에 동적으로 저장.
- `if~elif~else` 연속 조건문으로 A~F 학점 5단계 판별.
- `and` 논리 연산자로 **우수 과목(85점 이상 AND F학점 아님)** 조건 구현.
- `not` 연산자로 우수 과목이 없을 때 알림 출력.
- 과목명에 `'q'` 입력 시 `break`로 루프 조기 종료.
- `+=` 복합 대입 연산자로 점수 누적 계산.
- 리스트 메소드 6종 활용: `append`, `insert`, `remove`, `len()`, `sum()`, `max()`

| 요구 항목 | 구현 내용 |
|-----------|-----------|
| 변수 5개 이상 (str/int/float) | student_name(str), subject_count(int), last_semester_avg(float), total_score(float), excellent_count(int) 등 8개 |
| for + append 리스트 입력 | `for i in range(subject_count)` + `append()` 3회 |
| 리스트 메소드 4종 이상 | append / insert / remove / len · sum · max |
| if~elif~else | 학점 판별 5단계 |
| 중첩/독립 if | 우수 배지, F학점 경고 (출력 루프 내) |
| 관계 연산자 | `>=`, `!=` |
| 논리 연산자 | `and`, `not` |
| 복합 대입 연산자 | `total_score += score` |
| break | 과목명 `'q'` 입력 시 조기 종료 |

### 🤖 AI 파트너십 과정

#### 내용 1: 리스트 구조 설계 논의
- **프롬프트 요약**: "과목명, 점수, 학점을 저장하는 리스트를 어떻게 구성하면 나중에 딕셔너리로 확장하기 좋을까? 2차 과제 요구사항에 맞게 설계해줘."
- **적용 내용**: 과목명/점수/학점을 병렬 리스트로 분리하여 인덱스 기반 매핑 구조로 설계함. 4차에서 딕셔너리 전환이 쉬운 구조를 미리 확보함.

#### 내용 2: break / continue 활용법 논의
- **프롬프트 요약**: "2차 과제 평가에 break가 1회 이상 필요한데, 학습 도우미에서 자연스럽게 쓸 수 있는 상황을 제안해줘."
- **적용 내용**: 과목명 입력 시 `'q'`를 입력하면 루프를 조기 종료하는 `break` 로직 추가. 유효하지 않은 점수는 0점 처리 후 진행.

### 🔧 Troubleshooting & 기술 회고

1. **문제 1**: `sorted()`와 `sort()`의 차이를 몰랐음.
   - **해결**: `sort()`는 원본 리스트를 변경, `sorted()`는 새 리스트 반환. 원본 인덱스 매핑 유지를 위해 `sorted()`를 활용함.
2. **문제 2**: `sum(scores)`로 바로 계산하면 되는데 왜 `total_score += score`도 쓰는지 헷갈렸음.
   - **해결**: `+=` 복합 대입 연산자 사용 요건을 충족하기 위한 의도적 설계. `sum()`은 최종 검증용으로 병행 활용.
